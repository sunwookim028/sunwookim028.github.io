    Glossary¶

    One-line definitions of every domain-specific term used in this project. For reviewers without MLIR or NPU background; terms appear throughout isa/, sim/, compiler/, and docs/.

    Hardware model¶

    Term Definition
    NPU Neural Processing Unit - the custom AI accelerator this project targets.
    DIM Systolic-array tile dimension; DIM=8. All matmul tiles are DIM×DIM. (mininpu/hwparams.py)
    SPM Scratchpad Memory - bf16 dtype view of the on-chip pool, SPM_ROWS(=2048) × DIM; holds kernel inputs loaded from DRAM. (mininpu/hwparams.py)
    ACC Accumulator file - fp32 dtype view of the same on-chip pool, ACC_ROWS(=1024) × DIM; receives matmul output and intermediate activations. (mininpu/hwparams.py)
    VREG Vector Register File - VREG_COUNT(=8) registers × VREG_SIZE(=8) fp32 lanes; holds row-reduced scalars (rowmax, rowsum, etc.). (mininpu/hwparams.py)
    DRAM External main memory; 64-bit byte-addressed; sparse-backed in the simulator. (mininpu/iss/dram.py)
    hardware geometry A single fixed profile: DIM=8, one 32 KiB unified on-chip pool (SPM_ROWS=2048 bf16 / ACC_ROWS=1024 fp32 views), VREG_COUNT=8 × VREG_SIZE=8. The former NPU_PROFILE prod/tutorial dual-profile mechanism has been removed; the SSOT is mininpu/hwparams.py.
    bf16 / fp32 Storage vs accumulator precision. On-chip SPM storage is bf16 (STORAGE_FORMAT in mininpu/hwparams.py); ACC uses fp32 to avoid precision loss during accumulation.
    systolic array Hardware grid of DIM×DIM MAC units that executes matmul.tile in a single clock cycle.
    literal pool Assembler section holding wide constants (e.g., DRAM addresses) referenced by instructions as PC-relative offsets. (isa/npu-isa.md, "Why a fixed 64-bit word + literal pool")

    ISA mechanics¶

    Term Definition
    handle Symbolic name (%foo) for an SPM, ACC, or VREG buffer in .s assembly; the allocator maps it to a physical row/register. (isa/npu-isa.md, "Worked example - physical assembly form")
    bank Physical storage class: one of spm, acc, vreg. Each has a separate capacity and allocator. (mininpu/asm/_alloc/allocator.py)
    tile A DIM×DIM (or BLOCK_M×BLOCK_N) unit of work - the granularity at which matmul, DMA, and attention ops execute.
    opcode 8-bit field in the 64-bit fixed-width ISA instruction selecting the operation. (isa/npu-isa.md, "Why a fixed 64-bit word + literal pool"; mininpu/isa/opcodes.py)
    macro op CISC-style ISA op that expands to many hardware micro-ops, e.g., matmul.tile drives the systolic array for a full tile.
    iv / induction variable The loop counter of a loop.begin … loop.end nest; used in DMA address expressions. (isa/npu-isa.md, "Why loop bounds are literal-only (D4)")
    multi-iv DMA DMA address of the form base + Σᵢ ivᵢ·strideᵢ, supporting tiled DRAM traversal in a single instruction. Deferred to v1.3+ (MultiLinearAddr, D2b); not shipped in the frozen v1.x ISA, which caps iv/iv_stride arity at 1. (isa/npu-isa.md, "Known gaps (not yet implemented)")
    causal flag matmul.tile attribute that post-masks the upper triangle of the output to −∞ (FlashAttention causal self-attention).
    accumulate flag matmul.tile attribute that adds into an existing ACC buffer rather than zeroing it first.
    transB flag matmul.tile attribute that transposes operand B before multiplication (key projection in attention).
    flush Pipeline fence instruction; required before any host-visible DRAM store to ensure write ordering. (isa/npu-isa.md, "Why flush is the only ordering primitive (D9)")
    online softmax Algorithm computing softmax incrementally over tiles without materialising the full attention matrix; uses vec.rowmax, vec.exp2, vec.rowsum. (isa/npu-isa.md, "FlashAttention forward - annotated assembly (worked example)")
    row broadcast Expanding a VREG row-vector (shape [M]) across all columns of a 2D ACC tile (shape [M, N]).
    col broadcast Expanding a VREG column-vector (shape [N]) across all rows of a 2D ACC tile. Used in RoPE/RMSNorm.
    iter_arg MLIR scf.for loop-carried value - becomes an in-place handle alias in the NPU translator. (NPUToAsm.cpp, translateOp loop handling)

    MLIR concepts¶

    These are standard MLIR/LLVM terms; definitions here are purposely simplified for reviewers unfamiliar with the framework.

    Term Definition
    MLIR Multi-Level Intermediate Representation - LLVM's extensible compiler IR framework. All npu-opt/npu-translate transforms operate on MLIR.
    dialect A namespaced set of ops in MLIR, e.g., npu, linalg, scf, memref. Dialects are registered independently. (compiler/include/NPU/NPU.td)
    op / Operation A single MLIR instruction with typed operands, attributes, and result values.
    IR Intermediate Representation - the in-memory (and textual .mlir) program form that passes transform.
    Pass A self-contained IR transformation registered with a flag, e.g., --linalg-to-npu. (LinalgToNPUPass in LinalgToNPU.cpp)
    lowering Converting ops in a higher-level dialect to equivalent ops in a lower-level dialect, e.g., linalg → npu.
    linalg MLIR's linear-algebra dialect: provides linalg.matmul, linalg.reduce, linalg.fill, and the generic linalg.generic.
    linalg.generic Catch-all op encoding an arbitrary perfectly-nested loop with a body region; triton-shared-opt emits many of these.
    linalg.matmul Named matmul op C += A @ B (memref operands after bufferisation). Lowered by MatmulLowering.
    linalg.matmul_transpose_b C += A @ B^T; lowered by MatmulTransposeBLowering.
    linalg.fill Fills a buffer with a scalar; lowered to vec_zero_acc (zero) or vec_movi (constant).
    linalg.reduce Reduces along one axis with a body op; maximumf→vec_rowmax, addf→vec_rowsum.
    scf Structured Control Flow dialect: scf.for (counted loop), scf.yield (terminator).
    memref A typed reference to a memory region (pointer + shape + layout), analogous to a C array pointer with rank info.
    bufferization The compiler pass --one-shot-bufferize that converts tensor-valued SSA ops into memref (pointer-based) form. Required before --linalg-to-npu.
    DPS Destination-Passing Style - linalg ops where the output buffer is an explicit operand (getDpsInits()).
    OpRewritePattern MLIR base class for pattern-driven IR rewrites: subclasses override matchAndRewrite(). (LinalgToNPU.cpp:97)
    AffineMap An affine function from loop indices to operand dimensions, used in linalg.generic indexing maps.
    indexing maps Per-operand AffineMap list on linalg.generic declaring which loop dimensions each operand iterates over.
    iterator types Per-loop-dimension annotation: parallel (independent iterations) or reduction (must be combined).
    dyn_cast MLIR/LLVM RTTI downcast - returns nullptr/failure if the type doesn't match. Pervasive in both compiler files.
    lit / FileCheck LLVM/MLIR unit test tooling: lit runs test files, FileCheck pattern-matches output. FileCheck tests exist under compiler/test/ (Conversion/LinalgToNPU/, Translation/NPUToAsm/) and run via make lit-test.
    arith MLIR arithmetic dialect: arith.addf, arith.mulf, arith.truncf, arith.extf, arith.select, etc.
    math MLIR math dialect: math.exp2, math.sqrt, math.rsqrt.
    TTIR Triton IR - the IR form produced by @triton.jit compilation before lowering to linalg. (mininpu_compiler/compile.py:17)

    Triton concepts¶

    Term Definition
    Triton Open-source Python-embedded GPU kernel DSL; this project compiles Triton kernels targeting NPU instead of GPU.
    @triton.jit Decorator marking a Triton kernel; the decorated function is compiled by triton-shared-opt to linalg+scf MLIR.
    tl.constexpr Compile-time constant kernel parameter; specialised away during kernel compilation (like C++ template args).
    BLOCK_M / BLOCK_N / BLOCK_K Per-kernel tile dimensions declared as tl.constexpr; must equal DIM (or a multiple) for the NPU backend.
    pid / program_id Grid coordinate of the current kernel instance - analogous to CUDA's blockIdx. (tl.program_id(axis))
    grid N-D launch space; each grid point runs one kernel instance.
    atom kernel A minimal isolated Triton kernel covering one compute primitive (matmul, softmax row, RMSNorm, etc.). The 9 validated kernels in test/e2e/triton_validate/kernels/ (matmul, fa_fwd_causal_gqa, fa_fwd_noncausal, rmsnorm, rmsnorm_multitile, swiglu, rope, kv_cache, decode_attn) are atom kernels; gptq_matmul is excluded, it bypasses LinalgToNPU via a Builder-based program.
    triton-shared-opt Third-party MLIR tool (Microsoft/triton-shared) that lowers Triton IR to linalg+scf MLIR. First stage of the NPU pipeline.

    AI / model terms¶

    Term Definition
    GQA Grouped-Query Attention - H_Q query heads share H_KV key/value heads (H_Q > H_KV); reduces KV-cache size.
    FA / FlashAttention IO-aware tiled attention algorithm using online softmax to avoid materialising the full N×N attention matrix.
    RMSNorm Root-Mean-Square LayerNorm: normalises by 1/sqrt(mean(x²)) rather than mean and variance.
    RoPE Rotary Position Embedding - encodes position by rotating query/key pairs in 2D subspaces.
    SwiGLU Swish-Gated Linear Unit: silu(gate) * up; used as the FFN activation in LLaMA-2.
    KV-cache Per-token cached key/value tensors enabling O(1) decode steps in autoregressive generation.
    residual Skip-connection add in a transformer block; adds layer input to layer output.
    lm_head Final linear projection from hidden dimension to vocabulary logits; a large GEMM.
    greedy decode Token-generation strategy that selects the highest-probability token at each step (argmax).
    prefill Processing the full input prompt in one forward pass, producing the first output token and filling the KV-cache.
    decode-step Autoregressive generation of one token at a time using the cached KV entries from prefill.
    TinyLlama-1.1B 22-layer LLaMA-2 variant with 1.1 B parameters; the end-to-end reference model for M4c validation.
    sm_scale Softmax scaling factor 1/√D applied to attention scores before the online-softmax reduction.

    Project-specific¶

    Term Definition
    snapshot test Test that starts from a checked-in .mlir file at a specific pipeline stage and validates numerical output; independent of the Triton frontend. (test/e2e/pipeline/)
    sim-test make sim-test - runs the sim unit-test suite, no compiler binary needed.
    e2e-test make e2e-test - runs pipeline snapshot + numerical-correctness tests; requires npu-opt + npu-translate.
    triton-test make triton-test - runs Triton atom-kernel tests through the full pipeline; requires triton-shared-opt too.
    npu-opt MLIR optimiser tool for the npu dialect, with --linalg-to-npu pass; built to build/compiler/bin/npu-opt.
    npu-translate MLIR translation tool with --to-asm; emits .s text assembly; built to build/compiler/bin/npu-translate.
    DRAMAllocator Python allocator managing static (weights) and transient (activations) DRAM address arenas. (mininpu/iss/dram.py)
    liveness analysis Computes the first and last PC at which each handle is live; drives register allocation. (mininpu/asm/_alloc/liveness.py)
    first-fit Allocation policy: scan free intervals in order and pick the first one that fits. Used by RowAllocator.
    linear-scan Register-allocation algorithm: allocate in liveness order, free on last use. Used by both RowAllocator and RegisterAllocator.
    handle alias When a consumer op can reuse a producer's physical row in-place, the translator maps both handles to the same slot. (NPUToAsm.cpp, handles map in translateOp)
    SiLU chain Translator side-table tracking a multi-op SwiGLU sequence (sigmoid → negate → exp2 → add1 → div → mul) to fold it into one vec.silu emit. (NPUToAsm.cpp, siluSrcOf)
    scaleAddOf Translator side-table tracking an RMSNorm reciprocal-scale sequence to fold it into vec.fma.imm. (NPUToAsm.cpp, scaleAddOf)
    sqrtMarked Translator tag for a math.sqrt result that is about to be divided into 1 - fused into vec.rsqrt. (NPUToAsm.cpp, sqrtMarked)
