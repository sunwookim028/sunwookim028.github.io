Locked

Current design

Concurrency model

  • In-order issue, 1 instruction/cycle.
  • No producer-busy scoreboard; no hardware interlock.
  • All hazard avoidance is static compiler scheduling.
  • flush.slot is the only runtime sync primitive.
  • Nonblocking set = {mxu.matmul, mxu.load.w, mxu.acc.init, async-DMA}; everything else is blocking.

Distributed per-unit AGU

  • Each unit has its own address-generation unit (AGU).
  • Address = base + iv·stride + march.
  • base = compile-baked immediate + GRID_LINEAR launch patch (not a CSR).
  • iv (induction variable) is broadcast from the tile sequencer.

MXU

  • 8×8 weight-stationary systolic array.
  • bf16 × bf16 → fp32 accumulate.
  • 2 weight buffers (active / inactive double-buffer).
  • X streams from VREG, cast fp32 → bf16 at the PE input (no separate X input buffer).
  • Output → ACC.
  • RS-hybrid (row-stationary hybrid) dataflow.

VPU

  • Fixed-depth streaming pipeline, 8 lanes.
  • Function-select {reduce | lut | simd}.
  • D_VPU = 12 (gelu-driven, N-invariant, synth-tunable localparam).
  • Uniform depth across all functions.

VREG

  • 32 × [8×8] fp32 = 8 KiB.
  • Banked per-column, 2 read + 2 write ports (R_mxu / R_vpu / W_mxu / W_vpu).
  • Realized via bank-parallelism + static conflict-free scheduling (not a true multiport).

ACC

  • [8×8] fp32 × depth 2 = 512 B.
  • 8 per-column banks.
  • Bank is implicit (v1 uses bank 0).

SPAD

  • bf16 storage.
  • Separate, 0-based on-chip address space (like CUDA .shared).

MXU || VPU pipeline and roofline

  • 1-tile software pipeline overlapping MXU and VPU: ~22% overlap gain in attention (3:1 ratio); moot in MLP (40:1 ratio).
  • Roofline: MLP / matmul-heavy blocks are MXU-bound.
  • Roofline: devmem tiny matmul is DRAM-BW-bound - the 4 B/cy HP-port wall vs the 32 B/cy internal DM bus.

Open items

4-stage pipeline formalization (Fetch → Decode → Execute → Writeback)

  • Writeback targets VREG or ACC (matmul / acc.store write ACC, not VREG) - final stage naming / routing is TBD.
  • Exact issue / stall rules per op class, expressed in pipeline terms.
  • Whether a separate operand-read (VREG read) stage exists, and its latency.

Per-op Execute latencies

  • Must be sim == synth verified.

Remaining structural questions
