Live sequencer design

  • Reused as separate modules. pc.sv, decoder.sv, pc_iram.sv and loop_ctrl.sv stay separate modules, sequenced by the thin exposed-pipeline front-end: 1 instr/cycle, in-order, valid-shift, no monolithic multi-state FSM.
  • No datapath FSM. A valid-shift exposed pipeline: 1 instr/cycle, in-order, with no per-op micro-state machine.
  • Hazard resolution = NONE. Static schedule + fixed, data-independent FU latencies (an architecture invariant) make scoreboard-free issue safe: no producer-busy scoreboard, no WAIT_MXU / SYNC_MXU wait-states.
  • Only runtime sync. flush.slot spins until DMU async-slot-done; halt drains all DMA then raises done. The DMU is the sole unit with variable completion; everyone else is fixed-latency.
  • Distributed AGU. No central address generator: each of MXU / VPU / DMU / LSU receives {base,stride,count,flags,iv} + a start pulse and computes base + bo[ch] locally with an adder (no multiplier); the bo[8] base-offset accumulators live in loop_ctrl.sv (bo[ch] += stride at loop.end). iv is broadcast to all units.
  • Launch. Entry PC is fixed at IRAM[0]. The CP loads I_bram, pulses compute_start, and reads done / instr_ready.
