MXU design notes (locked)

  • X source = VREG. X is burst-read from VREG (fp32) via the R_mxu read port; SPAD is weight-only (plus the acc.init bias row).
  • fp32-X into bf16 PEs = CAST at the VREG read port (LOCKED). An explicit 8-lane fp32 → bf16 CAST sits at the R_mxu read port, upstream of the bf16 x_matrix staging buffer (not at the systolic-array west edge and not at the PE input). The PEs receive bf16 already staged in x_matrix and multiply bf16×bf16; accumulation stays fp32.
  • x_matrix = bf16 X staging buffer. S_LOAD_X fills the N×N buffer from the post-cast read port in exactly N=8 cycles, then unconditionally enters S_RUN, which streams the west edge out of x_matrix. VREG is therefore touched only for that bounded N-cycle burst: that temporal separation (not banking) is what avoids VREG port conflicts, and it is what the compiler scheduling rule relies on.
  • Output drains to ACC. Each systolic column drains its partial sums into its own ACC bank; acc.store is the only path back to VREG (fp32 lossless, driven by the LSU).
  • ACC = 8 banks × depth 2. One bank per column, double-buffered fp32, MXU-owned. accum folds K-tiles into the same ACC (serialized); causal masks the upper triangle. FA output O lives in VREG, so ACC stays depth 2.
  • mxu.acc.init. Broadcast-preloads a SPAD bias row into ALL rows of ACC before a matmul, so matmul accumulates onto bias.
  • Weight double-buffer. mxu.load.w fills the INACTIVE buffer while a matmul runs on the ACTIVE one; a matmul start implicitly PROMOTES inactive → active. transB reverses weight-row load order. Nonblocking; fixed latency = N-fill(8) + K-stream.
