    GPT-2 Kernel Gantt Figures: Generator + Output¶

    The gpt2_*.html files in this directory are cycle-level Gantt charts, one per GPT-2 kernel, each a self-contained inline-SVG standalone HTML styled like fig9_tile_gantt.html. They are generated, not hand-drawn.

    _gen_gpt2.py beside them is the source of truth. Each gpt2_<kernel>.html is a byte-for-byte product of running that script: it was regenerated and diffed against the committed HTML, and every landed figure matched exactly. This is the same source/output discipline the TikZ family follows (tikz/*.tex rendered via make -C docs figures): edit the source, re-render, never hand-edit the output HTML, or the next run silently reverts you.

    This note carries no cycle numbers on purpose. The numbers live only in _gen_gpt2.py (module constants + per-spec literals) and in the rendered HTML. Keeping them out of here means there is one place to change when they are finalized, and no stale copy to drift.

    How they are generated¶

    Requires only Python 3 (standard library; no third-party deps). Run it from this directory; it writes each figure next to itself.

    cd docs/mkdocs/figures
python3 _gen_gpt2.py

    It prints one wrote line per figure and one HELD line for any spec that is withheld (see below). To change a figure, edit the relevant spec in _gen_gpt2.py and re-run; do not touch the .html.

    What is HELD, and why¶

    _gen_gpt2.py carries a spec for every GPT-2 kernel, but one is deliberately not written to disk and not committed:

    • gpt2_gelu models gelu as a single fused latency-serial op. Owner ruling #385 section 5 decomposes gelu under ISA 0.5.0, so that model, and every roll-up that depends on it, is known-wrong. The spec is retained in the generator only as the section skeleton for whoever authors Doc 5; it is listed in the generator's HELD set and skipped on every run. Remove it from HELD only after the spec is recomputed under the decomposition.

    No other kernel sums a gelu cost, so the rest are unaffected: gpt2_cproj names gelu as the separate next kernel but does not add its cycles.

    Section-order map (reference for Doc 5, no numbers)¶

    The generator's spec order is the intended kernel-section order for the future Doc 5 page. Authoring that prose page (with numbers) is gated on the gelu ruling and is out of scope here; this is only the skeleton:

    1. gpt2_qkv_proj - attn QKV projection (MXU GEMM + bias)
    2. gpt2_qk_scores - attn Q @ K^T scores (scaled, causal)
    3. gpt2_softmax - attn row-softmax chain (VREG-only)
    4. gpt2_layernorm - LayerNorm stats + affine
    5. gpt2_gelu - MLP gelu activation (HELD, see above)
    6. gpt2_cproj - MLP down-projection GEMM
    7. gpt2_residual - residual add
    8. gpt2_lmhead - vocab projection + flush/halt tail

    These figures are unreferenced assets (no topic page embeds them yet), so they do not affect the mkdocs nav.
