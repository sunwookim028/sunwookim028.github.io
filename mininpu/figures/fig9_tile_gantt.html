mininpu tile-loop pipeline Gantt - FIG. 9

one (m,n) block of the 32x32 GELU worked example (ISA 0.5.0) · N=8 · weight double-buffer ping-pong · ACC depth-2 tail overlap · steady state = 1 matmul / N=8 cy, load.w fully hidden

SEQ issue / control (ctrl) MXU matmul stream / ACC accumulate (mxu) load.w / LSU acc.store · vreg.store (mem) VPU gelu (vpu) next-block matmul (ACC depth-2 overlap)

Steady state is 1 matmul / N=8 cy: while one weight buffer streams a matmul, the other loads the next tile's weights (load.w ~ N=8), so load.w is fully hidden. Each matmul result reaches ACC after the systolic fill 2N-1 + PE_LATENCY = 15 + 2 = 17. The ACC depth-2 double-buffer lets block j's drain / store tail (acc.store to gelu to vreg.store) overlap block j+1's matmul (dashed bar). The gelu here is vpu.lut, which is latency-serial (each row drains the full D_vpu = 23 before the next issues, not freely row-pipelined like the single-source SIMD ops). It is hidden only because the ACC depth-2 buffer overlaps this acc.store to gelu to vreg.store tail (plus the LSU = 10 latency) with block j+1's matmul, so on an MXU-bound tile-loop it stays off the throughput critical path.
