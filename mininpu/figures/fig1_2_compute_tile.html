Concurrency (130‖140‖160) = decoupled execution: engines self-sequence behind start/done (214); Tile sequencer does not block; no producer-busy scoreboard - fixed, data-independent FU latency is statically scheduled by the compiler, and the only runtime sync is flush.slot + halt-drain. Locked concurrency: MXU ‖ VPU 1-tile software pipeline over the shared VREG (2r1w async RF; conflict-free by temporal separation - the compiler schedules foreground VPU/LSU VREG access outside the short MXU X-read burst). No VLIW. Address base = compile-baked immediate + D4 launch patch (no CSR).

Design notes

  • Distributed AGU: MXU / VPU / LSU each own base + bo[ch] (adder, no multiplier) (137 / 152); no central AGU. DMU (160) is a black box behind SPAD - its AGU / slots (162) are not detailed here.
  • Tile sequencer (110): pc + decoder + pc_iram + loop_ctrl as separate modules + dispatch (iv_reg + descriptor); engine micro-FSMs live in the engines.
  • Launch I/F: CP (190) DMAs the binary to IRAM[0] (220), start (222), done (224).
  • mxu.acc.init bias-preload (204) into ACC (8 col-banks × depth 2, 138).
  • VREG (142) = 32 × [8×8] fp32 (8 KiB), 2r1w async whole-tile-row RF; VREG→MXU stream-read with fp32→bf16 cast at the MXU X-input (206, RS) - bf16×bf16 PE, fp32 accum.
  • acc.store (210) ACC→VREG (LSU); vreg.store VREG→SPAD.

No scoreboard

No producer-busy scoreboard, no HW hazard interlock. Fixed-latency (MXU/VPU/reg-move) is statically scheduled by the compiler (recompile-per-config accepted). The only runtime sync is the DMA fence.

compiler staticall fixed-latency deps (pad / fill NOPs)
flush.slotDMA completion (slot-done[0..1]) - only runtime sync

Exposed pipeline

  • Correctness from the instruction stream alone; latency owned by the compiler. Binaries recompiled per config (DIM / pipe depth).
  • "CISC" here = compiler-side macros to simplify its instruction selection - a compiler convenience, no hardware / ISA change. IRAM density is out of scope now.
  • The sequencer stalls only on flush.slot / halt-drain; hazard resolution is entirely static (no register scoreboard).
