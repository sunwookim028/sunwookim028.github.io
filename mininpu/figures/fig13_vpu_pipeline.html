Reference - FIG. 13 (RTL-verified, vpu_ctrl.sv / vpu_gelu_unit.sv)

  • D_vpu = 23, uniform issue-to-write for every function (stage_cnt terminal). gelu (deepest) sets it; shorter functions pad. Source: vpu_ctrl.sv LOCKED-depth block.
  • gelu ~22 stages: x2 @s3, x3 @s6, c3x3 @s9, poly @s12, z @s15, tanh @s16, (1+t) @s19 & half_x @s19, out @s22; + 1 BRAM-read cycle = D_vpu 23. Each fp_mul/fp_add is L=2 + a stage flop (3 stages, clean result). Source: vpu_gelu_unit.sv.
  • exp2 = 5: input @ cycle 1, result @ cycle 6 (BRAM 2^x chain; the embedded u_f_sub is fp_add L=2). Source: fp_exp2_bram.sv, vpu_ctrl stage schedule.
  • reduce-tree ADD = 3*DEPTH = 9 (DEPTH = log2(N) = 3): 2*DEPTH fp_add internal FFs + DEPTH inter-level FFs. MAX tree is DEPTH only. Source: fp_reduce_tree.sv.
  • Row-pipelining is partial: only the 5 single-source SIMD ops stream 1 row/cy (VPU_STREAM); everything else is 1 row / D_vpu. gelu is deliberately NOT row-pipelined.
