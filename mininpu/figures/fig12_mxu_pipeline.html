Reference - FIG. 12 (RTL-verified, mxu.sv / pe.sv)

  • PE MAC: fp_mul combinational (1 DSP48E2) -> mult_out_reg (Fix 10, 1 FF) -> fp_add accumulate LATENCY=PE_LATENCY=2. pe_psum_out is PE_LATENCY+2 cy from pe_valid_in; pe_valid_out leads it by 1. Source: pe.sv.
  • SKEW = PE_LATENCY + 1 = 3 (mxu.sv localparam). X-row r streams from phase 2N + SKEW·r; switch at 2N-1 = 15. SKEW_SPAN = (SKEW-1)(N-1) = 14.
  • PE_LAT = 2·PE_LATENCY + 1 = 5 (mxu.sv drain-margin localparam). S_RUN spans 5N + PE_LAT + SKEW_SPAN = 40 + 5 + 14 = 59.
  • Accumulator: mxu_accumulator RMW via fp_add (ACCUM_FORMAT) with write priority clear > acc_load > causal-fold > systolic-capture; the causal-fold add settles in S_CAUSAL_WAIT2. A 200 MHz timing-close pass may add +1 cy to this path (PENDING).
  • Throughput is unaffected by SKEW: the psum path is pure feed-forward (north->south), so only fill/drain grows; steady state stays 1 matmul / N=8 cy.
