Δ redesign vs current RTL

  • Real pipeline, latencies honored. Today: combinational datapath + retrofitted registers (the “Fix 1-16” timing campaign); FSM stage_cnt hand-mirrors physical register depth (fragile sim/synth invariant, drift bugs). Redesign: sim == synth.
  • gelu pipelined. Today: single 129-logic-level combinational cone (#217, forces 50 MHz). Redesign: split into pipe stages inside s4 → target clock.
  • Cross-row streaming. Today: ZERO cross-row pipelining — one row fully drains before the next (16-56 cyc/tile). Redesign: 1 row/cycle streamed → 8 + D cyc.
  • VREG-only, W_vpu = 1 of 2 write ports. Today: separate BRAM Port-B writeback mux + in-place ACC_ROW ops (#303). Redesign: VPU FU drives 1 of the VREG's 2 write ports (W_vpu), ACC_ROW dropped; FU touches VREG only (cast is the sole memory path).
  • VREG banking LOCKED. Banked per-column RF: 8 col-slices (one per lane), each = that column across 32 regs × 8 rows (256 fp32 words). Each slice serves 2 rd (R_mxu X-stream, R_vpu lane) + 2 wr (W_mxu acc.store, W_vpu result), banked by reg-id into simple 1r1w banks; the static schedule keeps concurrent reads on different banks and concurrent writes on different banks (conflict-free) → 2r2w without a true 2r2w SRAM. Bank count synth-tuned. (was OPEN monolithic-vs-per-lane; now locked.)
  • MXU X-input cast (LOCKED). The R_mxu read streams VREG fp32; an 8-lane fp32→bf16 cast sits at the MXU X-stream input (bf16×bf16 PEs, fp32 accumulate — no PE widening). Shown in FIG.1; VREG stays fp32 (VPU format).
  • LUT ROMs PLANNED (not yet built). Today = 4 inline-initial ROMs (exp2 / rsqrt / tanh / recip). PLANNED redesign target: unify into fp_lut_rom.sv with externalized .mem tables, drop recip (= rsqrt²), and instantiate fp_max for vpu.simd {max}.
  • Function-select, not 3 datapaths. ISA vpu.reduce / vpu.lut / vpu.simd map to one opcode-selected pipeline (reduce tree | LUT stage | SIMD lane-ALU), not parallel units.
