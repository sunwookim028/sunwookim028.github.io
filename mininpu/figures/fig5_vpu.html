Design notes (locked)

  • Real pipeline, latencies honored. The FU is a real fixed-depth pipeline under sim == synth: the reservation depth equals the physical register depth.
  • gelu pipelined. The gelu tanh cone is split into registered pipe stages inside s4 to meet the target clock.
  • Partial cross-row streaming. The single-source SIMD ops stream at 1 row/cycle (~8 + D_vpu cyc for an 8-row tile); gelu / exp2 / two-source ops stay latency-serial (1 row / D_vpu).
  • VREG-only, single write port. The VPU FU shares the VREG's single write port (temporal separation with the LSU) and touches VREG only; the LSU is the sole SPAD↔VREG path.
  • VREG conflict model LOCKED. 2r1w async whole-tile-row RF (256-bit): 2 read + 1 write ports shared by the MXU X-stream, the VPU lane read, and the LSU. Conflict-freedom is by temporal separation, NOT banking: the MXU reads its X tile in a short ~N-cycle burst, and the compiler schedules foreground VPU/LSU VREG access outside that window. A 2nd write port is a sizing option only (tail sub-ops are dependency-serial).
  • MXU X-input cast (LOCKED). The R_mxu read streams VREG fp32; an 8-lane fp32→bf16 cast sits at the MXU X-stream input (bf16×bf16 PEs, fp32 accumulate, no PE widening). Shown in FIG.1; VREG stays fp32 (VPU format).
  • Function-select, not 3 datapaths. ISA vpu.reduce / vpu.lut / vpu.simd map to one opcode-selected pipeline (reduce tree | LUT stage | SIMD lane-ALU), not parallel units.
