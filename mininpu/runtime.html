    Runtime: The Host-to-Device API¶

    A CUDA-style host-to-device API for driving the NPU from a Python script: device selection, pinned host memory, memcpy, and kernel launch, all built on the mininpu.abi SSOT (mininpu/runtime/). This page covers the calls a host script uses, plus the board HAL types they hand back. Implementation details (the UIO/udmabuf discovery, the two-phase doorbell handshake, the alignment contract) are deferred to the module docstrings and are not repeated here.

    Public API¶

    mininpuSetDevice¶

    def mininpuSetDevice(uio_name: str) -> None

    Selects the NPU device by UIO node name and installs it as the process-wide register-window backend.

    Selecting a device does not touch host allocations: as in CUDA, pinned host memory is process-wide and survives cudaSetDevice.

    mininpuHostAlloc / mininpuHostReset¶

    def mininpuHostAlloc(nbytes: int, *, udmabuf_index: int | None = None) -> HostRegion

    Allocates a pinned host DMA region (bump allocator over a single lazily-opened udmabuf) usable directly as a mininpuMemcpy host operand. Each region is 32-byte aligned to match the COPY alignment contract, and non-overlapping with every other region it hands out. This is the single source of host regions: do not hand-carve HostRegion off open_device's buf (see open_device).

    v1's bump allocator has no free (per-region free is [v2]), so a long-running caller allocates its regions once up front and reuses them rather than allocating per iteration. mininpuMemcpy takes the transfer length explicitly and reads only the region's phys_addr, so a smaller transfer can reuse a larger region.

    udmabuf_index applies only to the call that actually opens the buffer (None = no preference = udmabuf0). The runtime caches exactly one udmabuf per process, so an explicit index naming a different buffer than the cached one raises RuntimeError rather than silently serving the cached one; call mininpuHostReset() first to switch. If open_device already ran, it opened the buffer and its index is the one in force.

    def mininpuHostReset() -> None

    Drops the cached udmabuf and resets the bump offset, for test isolation and re-running allocation from a clean state. Closes the buffer's mmap only if mininpuHostAlloc opened it itself; a buffer installed by open_device belongs to the caller holding it and is not closed.

    mininpuMemcpy¶

    def mininpuMemcpy(dst: HostOperand, src: HostOperand, nbytes: int, kind: str) -> None

    Synchronous host-to-device or device-to-host copy via the ABI COPY command (the v1 primitive).

    A HostOperand is either an int device address or a host buffer exposing phys_addr (plus optional sync_to_device / sync_from_device cache hooks): a HostRegion or a UdmabufBuffer both qualify. dst and src name opposite sides, selected by kind. Both addresses must be 32-byte aligned.

    mininpuMemcpyAsync¶

    def mininpuMemcpyAsync(
    dst: HostOperand, src: HostOperand, nbytes: int, kind: str, stream=None
)

    Post-v1 stub: raises NotImplementedError (async COPY is a [v2] concurrency feature).

    mininpuLaunchKernel¶

    def mininpuLaunchKernel(kbin_addr: int, kbin_size: int, *, grid: tuple | None = None) -> None

    Issues a LAUNCH command for a kernel binary already resident in device memory.

    MemcpyKind¶

    class MemcpyKind:
    HostToDevice = "HostToDevice"
    DeviceToHost = "DeviceToHost"

    Direction selector for mininpuMemcpy (the cudaMemcpyKind analog).

    Kernel / load_program / program_to_bytes¶

    class Kernel:
    def __init__(self, asm_path, *, kbin_addr: int): ...
    def program_bytes(self) -> bytes: ...
    def run(self, host_buffer) -> None: ...

    Binds a pre-assembled .asm file to a device address. program_bytes loads and packs its instructions; run stages them (COPY H2D) and issues the LAUNCH.

    def load_program(path) -> "np.ndarray"

    Reads a pre-assembled .asm text program into a uint64 instruction array.

    def program_to_bytes(instructions: Sequence[int]) -> bytes

    Packs an instruction array into little-endian bytes for staging via a COPY.

    Board HAL¶

    Board bring-up and the types the calls above accept and return. A host script normally needs only open_device plus mininpuHostAlloc; the rest are the backend seam that keeps the runtime testable without hardware.

    open_device¶

    def open_device(uio_name: str, *, udmabuf_index: int = 0) -> tuple[Mmio, UdmabufBuffer]

    All-in-one board bring-up used by board tests and diagnostics: opens the NPU AXI-Lite window over uio_name, installs it as the process-wide MMIO backend, opens a udmabuf, and installs it as the runtime's single cached host buffer. Returns (mmio, buf). mininpuSetDevice deliberately does less (no udmabuf).

    The returned buf is for buffer-level use: buf.size (capacity checks), buf.phys_addr (diagnostics), and whole-buffer sync_to_device / sync_from_device.

    Get regions from the allocator, not from buf

    mininpuHostAlloc is the single source of host regions. Do not hand-carve HostRegion(buf, off, words) off the returned buf: the allocator's bump cursor cannot see a manual region, and both start at offset 0, so a manual region and an allocator region land on the same phys_addr and silently clobber each other's DMA. The allocator already guarantees the non-overlap and the 32-byte CDMA alignment a hand-picked offset would have to get right.

    HostRegion¶

    class HostRegion:
    def __init__(self, parent: UdmabufBuffer, byte_off: int, words: int): ...
    phys_addr: int          # parent base + byte_off; the COPY host_phys operand
    view: "np.ndarray"      # uint32 view over that slice of the parent mmap

    A phys-addressed sub-view of one contiguous udmabuf, and the return type of mininpuHostAlloc. Satisfies the host-operand protocol mininpuMemcpy consumes (a phys_addr plus cache-sync hooks, which delegate to the parent buffer).

    Obtain one from mininpuHostAlloc, not by constructing it directly: the constructor takes a caller-chosen byte_off the allocator knows nothing about (see the open_device warning above).

    Mmio / MockMmio / UioMmio¶

    class Mmio(ABC):
    def read32(self, offset: int) -> int: ...
    def write32(self, offset: int, value: int) -> None: ...

class MockMmio(Mmio):
    def __init__(self, ready_after: int = 0, read_values: dict[int, int] | None = None): ...

class UioMmio(Mmio):
    @classmethod
    def open_by_name(cls, name: str, *, map_index: int = 0) -> "UioMmio": ...

    Mmio is the 32-bit register-access seam every command write goes through. MockMmio records writes and replays scripted reads for tests with no board; UioMmio mmaps the real AXI-Lite window via /dev/uioN, discovered by name.

    UdmabufBuffer¶

    class UdmabufBuffer:
    @classmethod
    def open(cls, index: int = 0, *, size: int | None = None,
             sysfs_dir=None, dev_path=None) -> "UdmabufBuffer": ...
    phys_addr: int
    size: int
    def sync_to_device(self) -> None: ...    # flush CPU cache -> DDR (H2D)
    def sync_from_device(self) -> None: ...  # invalidate CPU cache <- DDR (D2H)
    def close(self) -> None: ...

    A contiguous host DMA buffer backed by /dev/udmabufN, with its physical address read from sysfs. The parent that HostRegion carves from.

    CUDA correspondence¶

    mininpu call CUDA call
    mininpuSetDevice(uio_name) cudaSetDevice(int device)
    mininpuHostAlloc(nbytes) cudaHostAlloc / cudaMallocHost
    mininpuHostReset() (no direct analog; test/re-run helper)
    open_device(uio_name) (no direct analog; board bring-up primitive)
    mininpuMemcpy(dst, src, nbytes, kind) cudaMemcpy
    mininpuMemcpyAsync(...) cudaMemcpyAsync (not yet implemented)
    mininpuLaunchKernel(kbin_addr, kbin_size) kernel launch (<<<...>>> / cuLaunchKernel)
    MemcpyKind cudaMemcpyKind
