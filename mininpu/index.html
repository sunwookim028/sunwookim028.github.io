    mininpu: NPU IP and Triton to Silicon Flows¶

    mininpu · documentation

    A composable NPU IP template (MXU / VPU / tile / NoC) built as an IO-attached discrete accelerator, plus the host runtime that drives it and the compiler that targets it. FPGA is a test platform only; the design stays ASIC-targeting.

    • ISA 0.5.0 · 15 opcodes
    • geometry N=8
    • VREG 32 fp32 vector registers (flat 1D; as-built RTL still 2D [8x8])
    • on-chip SPAD 32 KiB
    • target GPT-2

    Hardware¶

    The NPU IP: an architectural spec, one microarchitecture implementing it, and the flows that build it.

    • Architectural spec: ISA Contract · ABI Register Map
    • Microarchitecture: Compute tile (overview, Sequencer, VPU, MXU, SPAD, DMU, Memory Tile ([v3]-deferred, RFC #158)) · GPT-2 demo · The uncore (the non-compute infra)
    • Implementation flows: ASIC flow (the primary path; both generator stacks target ASIC) · FPGA flow (test platform only, no FPGA-specific RTL) · both pages are placeholders that scope the flow rather than describe it

    Spec vs RTL as-built

    Spec: ISA 0.5.0 (frozen; amendments tracked in #381) · RTL as-built: 0.4.0 · cutover pending. A spec is not wrong because the RTL trails it; the pages above assert the 0.5.0 spec, and never claim the hardware implements it.

    Runtime¶

    The in-repo Host Runtime API (mininpu/runtime/): a CUDA-style host/device API for driving the NPU from a Python script, built on the mininpu.abi register-map SSOT. Device selection, pinned host allocation, synchronous and async memcpy, and kernel launch.

    Compiler¶

    The compiler is developed in a separate repository and published as its own doc site: mininpu-compiler, a minimal extensible compiler framework lowering to the NPU ISA.

    Labs¶

    The teaching path (lab0..5) builds the accelerator from the inside out, one subsystem per lab. The whole-device / host-in-the-world overview lives on Lab 0, not here.
