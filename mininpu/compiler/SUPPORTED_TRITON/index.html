    Supported Triton Constructs¶

    SSOT - compile surface. This file is the authoritative record of what the NPU compiler can and cannot lower from Triton. Peer to charter.md (roadmap/design SSOT) and the sibling mininpu/isa/opcodes.py (ISA SSOT). Defects/gaps are tracked as GitHub issues, not here.

    What the NPU compiler can and cannot lower from Triton source. Use this as a pre-flight checklist before writing a new kernel.

    1. Supported tl.* primitives¶

    These primitives are empirically validated through the 9 atom kernels in test/e2e/triton_validate/kernels/ (matmul, fa_fwd_causal_gqa, fa_fwd_noncausal, rmsnorm, rmsnorm_multitile, swiglu, rope, kv_cache, decode_attn; gptq_matmul is excluded, it bypasses LinalgToNPU via a Builder-based program). Any usage pattern found in those kernels compiles and produces correct output; other usage patterns are untested.

    Primitive Notes Validated by
    tl.program_id(axis) Grid coordinate; axes 0, 1, 2 all work all kernels
    tl.arange(0, N) Constant range; N must be tl.constexpr all kernels
    tl.constexpr parameters Compile-time specialisation all kernels
    tl.load(ptr, mask, other) mask/other supported; cache_modifier ignored all kernels
    tl.store(ptr, val, mask) mask supported all kernels
    tl.zeros((M, N), dtype) Initialises ACC tile matmul, fa, rmsnorm_mt
    tl.full((M, N), val, dtype) Constant fill fa
    tl.dot(a, b, acc=…) Both acc and non-acc forms matmul, fa
    tl.trans(a) Transposes a 2D block (key projection) fa
    tl.max(x, axis=1) Row-reduce max fa
    tl.sum(x, axis=1) Row-reduce sum fa, rmsnorm, rmsnorm_mt
    tl.maximum(a, b) Element-wise max (online-softmax update) fa
    tl.where(cond, x, y) Only the FA causal-mask pattern compiles (see §4) fa
    tl.math.exp2(x) Element-wise exp₂ fa, swiglu
    tl.sqrt(x) Element-wise sqrt (via math.sqrt in linalg) rmsnorm, rmsnorm_mt
    .to(tl.float16/.float32) Type cast (truncf / extf) all kernels

    Untested / known not to compile: tl.exp, tl.log, tl.minimum, tl.softmax, tl.reshape, tl.broadcast_to, tl.cat, tl.atomic_*, tl.cdiv, tl.dot without acc= and with non-constexpr K, tl.where outside the FA causal-mask pattern.

    2. LinalgToNPU rewrite patterns¶

    These 21 patterns in compiler/lib/Conversion/LinalgToNPU/LinalgToNPU.cpp convert named linalg ops and specific linalg.generic body shapes to npu.* dialect ops. Every op matched here will be correctly lowered to assembly.

    Matmul (M3)¶

    Pattern linalg op npu op File:line
    MatmulLowering linalg.matmul (C += A @ B) npu.matmul_tile :97
    MatmulTransposeBLowering linalg.matmul_transpose_b npu.matmul_tile{transB} :118

    accumulate flag is set when C is defined by memref.alloc (on-chip ACC), not when it is a function argument (DRAM). The distinction is purely structural.

    Fill / initialise (M3)¶

    Pattern linalg op npu op File:line
    FillLowering linalg.fill 2D f32 with zero npu.vec_zero_acc :138
    FillLowering linalg.fill 1D f32 with constant npu.vec_movi :138

    Reductions ACC→VREG (M3)¶

    Pattern linalg op npu op File:line
    ReduceMaxLowering linalg.reduce{maximumf} 2D→1D npu.vec_rowmax :175
    ReduceSumLowering linalg.reduce{addf} 2D→1D npu.vec_rowsum :206

    Only maximumf and addf reduction bodies are matched. minimumf, mulf, and other reducers fall through to the ad-hoc matcher (§3) or emit nothing.

    FlashAttention vector ops (M3) - ACC ↔ VREG¶

    These patterns lower the element-wise tile ops used in online-softmax.

    Pattern linalg.generic body npu op File:line
    VecMaximumLowering 1D×1D→1D maximumf npu.vec_maximum :235
    SubBcastLowering 2D − 1D(row) → 2D subf npu.vec_sub_bcast :260
    Exp2AccLowering 2D → 2D math.exp2 npu.vec_exp2 :285
    Exp2VregLowering 1D → 1D math.exp2 npu.vec_exp2_vreg :312
    ScaleBcastLowering 2D × 1D(row) → 2D mulf npu.vec_scale_bcast :335
    DivBcastLowering 2D / 1D(row) → 2D divf npu.vec_div_bcast :363
    SubVregLowering 1D − 1D → 1D subf npu.vec_sub_vreg :417
    FmaVregLowering 3×1D → 1D mulf+addf (FMA) npu.vec_fma :443
    CopyVregLowering memref.copy 1D f32 npu.vec_copy_vreg :481

    Casts (M3 + M4a)¶

    Pattern linalg.generic body npu op File:line
    TruncF32F16Lowering 2D → 2D arith.truncf f32→f16 npu.cast_f32_f16 :388
    ExtendF16F32Lowering 2D → 2D arith.extf f16→f32 npu.cast_f16_f32 :504

    Element-wise ACC ops (M4a - RMSNorm, SwiGLU, RoPE)¶

    Pattern linalg.generic body npu op File:line
    MulAccLowering 2D × 2D → 2D mulf npu.vec_mul_acc :532
    AddAccLowering 2D + 2D → 2D addf npu.vec_add_acc :557
    SubAccLowering 2D − 2D → 2D subf npu.vec_sub_acc :582
    ScaleColBcastLowering 2D × 1D(col) → 2D mulf npu.vec_scale_col_bcast :607
    RsqrtVregLowering 1D → 1D math.rsqrt npu.vec_rsqrt :636

    Note: SiluAccLowering is declared (:28) but not added to the pipeline (:657-660). The SwiGLU silu folding is handled ad-hoc in the translator (§3). This is tracked in #10.

    3. Ad-hoc linalg.generic matcher in NPUToAsm.cpp¶

    The translateOp() ad-hoc body matcher in compiler/lib/Translation/NPUToAsm/NPUToAsm.cpp matches a second set of linalg.generic bodies directly to assembly, bypassing the dialect layer. This was the historical entry-point before LinalgToNPU grew its pattern set; some patterns here predate their dialect equivalents. The split is tracked in

    10 as architectural debt. (Per-branch line numbers are¶

    omitted - the file has been split/grown; grep translateOp for the branch.)

    Body pattern Emitted as
    Zero-operand (pass-through) handle alias (no emit)
    arith.truncf f32→f16 cast.f32.f16
    arith.extf f16→f32 (2D) cast.f16.f32
    arith.extf f16→f32 (1D, VREG) cast.f16.f32.vreg
    arith.mulf with row-bcast vec.scale.bcast
    arith.mulf with col-bcast vec.scale.col.bcast
    SwiGLU: silu(gate)*up multi-op chain vec.silu (folded)
    arith.mulf 2D × constant vec.scale.imm
    arith.mulf 2D general vec.mul.acc
    arith.mulf 1D general vec.mul.vreg
    arith.subf 2D with row-bcast RHS vec.sub.bcast
    arith.subf 2D general vec.sub.acc
    arith.subf 1D general vec.sub.vreg
    arith.addf RMSNorm bias fold vec.fma.imm (folded)
    arith.addf 1D general vec.add.vreg
    arith.addf 2D general vec.add.acc
    RMSNorm: 1/sqrt(mean(x²)) chain vec.fma.imm + vec.rsqrt (folded)
    arith.divf 2D with row-bcast denom vec.div.bcast
    math.sqrt 1D marks sqrtMarked, no emit
    math.exp2 2D vec.exp2
    math.exp2 1D vec.exp2.vreg
    arith.select (FA causal mask) handle alias to qk (no emit)
    arith.maxnumf 1D vec.maximum
    Any other body silent no-op - see §4

    4. Supported linalg named ops and scf constructs¶

    linalg named ops matched¶

    Op Lowered to Where
    linalg.matmul npu.matmul_tile LinalgToNPU.cpp:97
    linalg.matmul_transpose_b npu.matmul_tile{transB} LinalgToNPU.cpp:118
    linalg.fill (2D f32 zero) npu.vec_zero_acc LinalgToNPU.cpp:138
    linalg.fill (1D f32 const) npu.vec_movi LinalgToNPU.cpp:138
    linalg.reduce{maximumf} 2D→1D npu.vec_rowmax LinalgToNPU.cpp:175
    linalg.reduce{addf} 2D→1D npu.vec_rowsum LinalgToNPU.cpp:206
    memref.copy 1D f32 npu.vec_copy_vreg LinalgToNPU.cpp:481

    Not matched: linalg.matmul_transpose_a, linalg.batch_matmul, linalg.conv_*, linalg.transpose, linalg.copy. Any linalg.reduce with a reducer other than maximumf or addf falls through.

    scf constructs¶

    Construct Behaviour
    scf.for with literal bounds Lowers to loop.begin … loop.end
    scf.for with outer_iv * N bound Supported; becomes hi=pid_m*BLOCK_N in assembly
    scf.for iter_args (loop-carried ACC/VREG) Handled via in-place handle alias (NPUToAsm.cpp, translateOp)
    scf.yield Implicit terminator; no emit
    scf.if Not supported - no match in either lowering file

    5. What does not compile¶

    Construct Reason
    Non-tile-aligned shapes (M % BLOCK_M ≠ 0) Host must pad; kernel assumes aligned shapes
    scf.if No match in LinalgToNPU or NPUToAsm
    tl.where outside the FA causal-mask pattern Generic arith.select silently aliases without emit
    linalg.batch_matmul, linalg.conv_* No pattern
    linalg.reduce{minimumf}, {mulf}, etc. No pattern for reducers other than maximumf/addf
    arith.maxnumf 2D Only 1D matched (NPUToAsm.cpp, translateOp)
    Multi-headed GQA with H_KV > 1 fa_fwd_causal_gqa.py:33 hardcodes H_KV=1
    tl.exp, tl.log, tl.minimum Not lowered to any ISA op

    6. Formerly-silent failure modes (now hard errors)¶

    These paths used to succeed while producing garbage. W9 turned every one into a hard error (emitOpError + failed translation), so npu-translate now fails loudly instead of emitting junk - this is what makes Gate 2 ("zero silent no-ops") enforceable. Symbol anchors below (line numbers omitted; the translator was split into NPUToAsm.cpp / PhysAllocPass.cpp / LinearAddrAnalysis.cpp).

    Where (file, symbol) Formerly-silent case Now
    NPUToAsm.cpp, translateOp Unknown linalg.generic body - was no emit Hard error
    PhysAllocPass.cpp, runPhysAlloc/linearScan Phys-alloc overflow - was silent base=0 → spm_row=0 Hard error
    LinearAddrAnalysis.cpp, buildRCAddrStr Unresolved DRAM/RC address - was literal "?" Hard error
    NPUToAsm.cpp, translateOp Unknown npu.* op - was ; TODO: <opname> Hard error
    LinalgToNPU.cpp Shape mismatch - returns failure() Op survives to translator, hard-errors there

    Tracked as closed (W9).
