    NPU ISA - "v1.x freeze" (non-normative narrative)¶

    Version axes - do not conflate. "v1.1/v1.2/v1.x freeze" here is a spec codename for this narrative's design-decision history, not the shipped package version. The package's machine-readable version is mininpu.isa.opcodes.ISA_VERSION (currently 0.4.0, pre-1.0), which is the only version number an importer should trust.

    Normative SSOT: mininpu/isa/opcodes.py. All normative ISA definitions - opcode values + per-opcode semantics, FIELDS_* bit layouts, the D7 flags map, capacity constants, the frozen DMA descriptor caps, and the Numerical Precision Contracts (tolerance tiers + per-opcode valid input domains + bit-exact list) - live exclusively in mininpu/isa/opcodes.py as structured module/section/function docstrings and comments adjacent to the constants and encoders they define. This document is non-normative: rationale, the annotated FlashAttention assembly walkthrough, worked examples, and historical narrative only. It contains no opcode-value table, no encoding bit-layout table, no flags map, no capacity numbers, and no precision-tolerance table - those were removed (W10, per the v1.x ISA freeze) so there is no hand-maintained duplication. If anything here conflicts with mininpu/isa/opcodes.py, opcodes.py wins; update it first.

    This structural single-SSOT consolidation is what makes the opcode-set / FLUSH-value divergence class (D3 / D6) impossible: there is exactly one place to change. (OQ-7: divergence is made impossible, not policed by an equality CI.)

    Changelog (historical narrative - kept verbatim for provenance): - v1.1 (M4a): vec.rsqrt, vec.silu, vec.mul/add/sub.acc, vec.scale.col.bcast, cast.f16.f32, matmul.tile causal flag, multi-iv DMA, iv-dependent loop bounds, NPU_PROFILE. (Note: multi-iv DMA and iv-dependent loop bounds were removed/deferred by the v1.x freeze below - D2b/D4. This line is historical.) - v1.2 (M4b): vec.fma.imm (0x27), vec.add.vreg (0x28), vec.mul.vreg (0x29), vec.copy.acc (0x2A), cast.f16.f32.vreg (0x32). - v1.x freeze: iv/iv_stride arity capped at 1 (D2b; MultiLinearAddr deferred to v1.3+); loop bounds are literal-only (D4; scaled/iv-derived bounds removed, per-pid_m bound set by host IRAM-patch); FLUSH moved into the control-group nibble (D6, was 0xFF); normative flags map adopted (D7); flush drains async DMA on hardware / sim no-op, flags[5]=async reserved (D9); SPM/ACC clarified as a logical dtype partition over one unified byte-pool (D1). All these landed normatively in mininpu/isa/opcodes.py; this bullet records the decision history. vec.copy.acc (0x2A) was subsequently removed (D3/OQ-5) - the compiler lowers it to vec.scale.imm × 1.0; this changelog line is left intact as provenance.

    Minimal hybrid ISA for a custom AI accelerator targeting tiled GEMM and online-softmax FlashAttention. Designed to be extensible toward CISC-style macro execution on RTL hardware. The exact encoding, opcode set, hardware model and numerical contracts are defined in mininpu/isa/opcodes.py (open that file to see both the encoding and its authoritative spec in one place); the material below explains why the design is shaped this way and how a representative kernel maps onto it.

    Design rationale¶

    Why a fixed 64-bit word + literal pool¶

    The instruction word is deliberately a single fixed-width 64-bit value with only three operand slots. This keeps the RTL decoder trivial (no variable-length fetch, no operand-count state machine) at the cost of two operand classes that do not fit inline: wide values (full DRAM byte addresses, dram_stride, fp32 immediates) and extra operands (instructions with more than three operands, e.g. vec.fma). Both spill to a per-program literal pool emitted after the program body; the operand slot then holds a pool index and a D7 IMM_OP* flag bit marks the indirection. fp32 immediates are stored bit-exact (IEEE-754 pattern) so no precision is lost crossing the 16-bit operand boundary. The exact field positions, flag-bit assignments and pool mechanics are normative in mininpu/isa/opcodes.py (FIELDS_INSTR, the flags-byte docstring, LiteralPool).

    Why DMA is kept in an inline form¶

    The DMA opcodes carry an inline (dst, dram_addr, mat_rows, mat_cols, dram_stride) form rather than exposing hardware descriptor registers directly. This keeps kernel assembly readable and lets the compiler reason about DMA shape symbolically; the adopted assembler.py (W6) lowers the inline form into RTL descriptor-config + trigger words at the npu-translate boundary. The frozen descriptor caps (mat_rows, mat_cols, dram_stride) are a hardware limit, enforced by the encoders in mininpu/isa/opcodes.py (they raise rather than silently truncate, which would corrupt the DMA shape).

    dram_stride=0 broadcast encoding: dram_stride=0 is a valid encoding for dma.load.spm (and the typed variants) meaning "broadcast": every row of the SPM tile is loaded from the same DRAM base address. The simulator implements this by repeating the single row mat_rows times. The primary use case is broadcasting a 1-row bias vector to BLOCK_M rows for a bias-add without requiring a dedicated opcode or host-side tensor replication.

    Why SPM and ACC are one physical pool (D1)¶

    "SPM" and "ACC" are not separate memories - they are the fp16 and fp32 views of one unified on-chip byte-addressed scratch. Keeping them logically distinct in the ISA preserves clean instruction semantics (a matmul writes "ACC", a cast writes "SPM") while the allocator treats the pool as a single byte-weighted budget. ACC_ROWS is therefore a convenience figure, not an independent hardware budget. This is a clarification only; instruction semantics are unchanged. The capacity numbers and the unified-pool sizes are normative in mininpu/isa/opcodes.py.

    Why loop bounds are literal-only (D4)¶

    Earlier drafts allowed iv-derived / scaled loop bounds so the causal FlashAttention phase-1 bound (pid_m × BLOCK_N) could be computed in hardware at loop entry. The freeze removes this: hardware does no multiply/address arithmetic at loop entry, the bound is a literal, and the host IRAM-patches the per-pid_m value before each grid launch via the patch table (a compiler↔abi.py contract; see mininpu/abi.py). This trades a tiny amount of host work for a much simpler sequencer and makes one program serve every grid launch. The literal-only loop encoding and the single-iv (arity-1, D2b) loop-body DMA address mode are normative in mininpu/isa/opcodes.py.

    Why flush is the only ordering primitive (D9)¶

    There is no async-DMA programming model in frozen v1.x, so a single global flush fence is the sole ordering primitive; flags[5]=async is reserved with no v1.x semantics. flush drains in-flight async DMA on hardware and is a no-op in the functional simulator. It must precede any off-chip store a host will read. The opcode value and these semantics are normative in mininpu/isa/opcodes.py.

    FlashAttention forward - annotated assembly (worked example)¶

    This is the canonical worked example showing how the opcode set composes into a real kernel. It is illustrative, not a normative encoding reference - every mnemonic below is defined normatively in mininpu/isa/opcodes.py.

    Causal variant (M4a canonical form)¶

    # Inputs:  Q[BLOCK_M, HEAD_DIM], K[N_CTX, HEAD_DIM], V[N_CTX, HEAD_DIM] fp16 in DRAM
# Output:  O[BLOCK_M, HEAD_DIM] fp16 in DRAM
# Params:  qk_scale = sm_scale * log2(e)  (base-2 exp trick)
# Grid:    outer loops over pid_m (M-blocks) and heads (not shown here)
#
# Two-phase structure:
#   Phase 1 - full blocks below the diagonal (n < pid_m * BLOCK_M): no mask
#   Phase 2 - diagonal block (n = pid_m * BLOCK_M):          causal=1 mask

    dma.load.spm    q,  Q_ADDR,  BLOCK_M, HEAD_DIM, HEAD_DIM*2
    vec.movi        mi, -inf,    BLOCK_M
    vec.movi        li,  1.0,    BLOCK_M
    vec.zero.acc    acc, BLOCK_M, HEAD_DIM

    ; ── Phase 1: non-diagonal K/V blocks ────────────────────────────────────
    ; upper bound is a literal placeholder (D4); the host IRAM-patches it to
    ; pid_m*BLOCK_N per grid launch via the abi.py patch table (FREEZE §10).
    loop.begin  n, 0, HI_PATCH, BLOCK_N        ; HI_PATCH = literal, host-patched
        dma.load.spm    k, K_ADDR  iv=n  iv_stride=HEAD_DIM*2  BLOCK_N, HEAD_DIM, HEAD_DIM*2
        dma.load.spm    v, V_ADDR  iv=n  iv_stride=HEAD_DIM*2  BLOCK_N, HEAD_DIM, HEAD_DIM*2

        matmul.tile     qk, q, k,  transB=1, accumulate=0      ; qk = q @ k^T
        vec.scale.imm   qk, qk, qk_scale

        vec.rowmax      mij_raw, qk
        vec.maximum     mij, mi, mij_raw

        vec.sub.bcast   p, qk, mij
        vec.exp2        p, p

        vec.rowsum      lij, p
        vec.sub.vreg    mdiff, mi, mij
        vec.exp2.vreg   alpha, mdiff

        vec.scale.bcast acc, acc, alpha
        cast.f32.f16    p_fp16, p
        matmul.tile     acc, p_fp16, v,  accumulate=1

        vec.fma         li, alpha, li, lij
        vec.copy.vreg   mi, mij
    loop.end n

    ; ── Phase 2: diagonal block with causal mask ─────────────────────────────
    dma.load.spm    k, K_ADDR  iv=pid_m  iv_stride=HEAD_DIM*2  BLOCK_N, HEAD_DIM, HEAD_DIM*2
    dma.load.spm    v, V_ADDR  iv=pid_m  iv_stride=HEAD_DIM*2  BLOCK_N, HEAD_DIM, HEAD_DIM*2

    matmul.tile     qk, q, k,  transB=1, accumulate=0, causal=1  ; upper-tri → -INF
    vec.scale.imm   qk, qk, qk_scale

    vec.rowmax      mij_raw, qk
    vec.maximum     mij, mi, mij_raw

    vec.sub.bcast   p, qk, mij
    vec.exp2        p, p

    vec.rowsum      lij, p
    vec.sub.vreg    mdiff, mi, mij
    vec.exp2.vreg   alpha, mdiff

    vec.scale.bcast acc, acc, alpha
    cast.f32.f16    p_fp16, p
    matmul.tile     acc, p_fp16, v,  accumulate=1

    vec.fma         li, alpha, li, lij

    ; ── Tail ─────────────────────────────────────────────────────────────────
    vec.div.bcast   acc_norm, acc, li
    cast.f32.f16    o_fp16, acc_norm
    dma.store.spm   O_ADDR, o_fp16
    flush

    The two-phase structure is the key composition idea: every non-diagonal K/V block runs the online-softmax update with no mask; only the single diagonal block sets matmul.tile causal=1, which masks the tile-local upper triangle to -INF post-multiply. The loop's upper bound is a literal placeholder that the host patches per pid_m launch (D4) - the program itself is launch-invariant.

    Worked example - physical assembly form¶

    The .s file format (M3.5c and later) carries both symbolic handles and physical bank indices. Physical kwargs are mandatory in compiler-emitted programs; parse_asm validates them against an independent allocator run and raises ValueError if compiler and sim disagree on placement:

    matmul.tile  r0  r1  r2  transB=1  accumulate=0  acc_row=0  spm_a_row=256  spm_b_row=512
vec.movi  r3  imm=-3.4028235e+38  rows=64  vreg=3

    Test harnesses use mininpu.iss.dram.DRAMAllocator to assign DRAM addresses within the partitioned address space (defined normatively in mininpu/isa/opcodes.py) rather than hard-coding values.

    Extensibility notes¶

    Completed¶

    • Triton frontend (M4a): triton-shared-opt --triton-to-linalg-experimental
    • extended LinalgToNPU patterns for the 7 M4a/M4b atom kernels (GEMM, FA-fwd causal GQA, RMSNorm, RMSNorm-multitile, SwiGLU, RoPE, KV-cache append).
    • Transformer-block composition (M4b): 7/7 Triton atom kernels pass on both prod and tutorial profiles. ISA v1.2 additions: vec.fma.imm, vec.add/mul.vreg, vec.copy.acc, cast.f16.f32.vreg (all now normative in mininpu/isa/opcodes.py; vec.copy.acc subsequently lowered, D3).
    • TinyLlama-1.1B e2e (M4c): 22-layer greedy decode; HF weight bridge (safetensors, bf16→fp16); token-exact match vs HuggingFace float32 confirmed.
    • Decode-step attention (N=1): a host-padded [1,D]→[BLOCK_M,D] kernel reusing the FA path, zero ISA change (per the v1.x freeze). Shipped as the decode_attn atom kernel and validated end-to-end (test_decode_attn).

    Planned¶

    • New scalar activation (GELU, LayerNorm): vec.gelu is reserved (opcode allocated in mininpu/isa/opcodes.py, unimplemented - OQ-8); activating it needs one encoder + one handlers.py entry. No other changes.
    • Backward pass: needs vec.scale.bcast, vec.rowsum, vec.exp2.vreg (already present) plus one new dma.atomic.add for gradient accumulation (deferred, no v1.x reservation - OQ-8).

    Known gaps (not yet implemented)¶

    1. Tile-aligned only - non-tile-aligned shapes require host-side padding; partial-tile masking is the host's responsibility.
    2. Binary encoding: the 64-bit fixed-width encoding is defined in mininpu/isa/opcodes.py; the compiler adopts assembler.py as the binary producer (W6 / D5 reversal - ISA known-gap #2 closes there).
    3. No address descriptor registers - loop-body DMA addresses use inline LinearAddr expressions; the adopted assembler.py lowers them to hardware descriptor + trigger words.
    4. No int8 / quantized mode - int8 opcodes are reserved-only in mininpu/isa/opcodes.py (D10); not wired into DMA or matmul.
    5. No async DMA / multi-channel barriers - only one global flush fence; flags[5]=async reserved (D9).
    6. No conditional control flow - only counted loop.begin/loop.end.
    7. Address space encoded per-opcode, not per-type - adding a fourth memory space requires a new opcode rather than a memref type attribute.
    8. No multi-tenant / process-id support.
    9. Causal mask is matmul-only - applied post-multiply inside matmul.tile; no standalone vector causal-mask op.
    10. Loop bounds are literal-only (D4) - per-pid_m variation is realized by host IRAM-patch of the literal (compiler↔abi.py patch-table contract, FREEZE §10), not by an ISA semantic. MultiLinearAddr (iv-list length > 1 in loop-body DMA) is deferred to v1.3+ (D2b).
