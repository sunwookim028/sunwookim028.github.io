Locked · sequencer fold

PC and decoder move inside the tile sequencer. The combined unit is the issue unit: IRAM 4096 x 64b + decoder. The separate pc.sv / decoder.sv / pc_iram.sv are folded in as reused submodules (structural fold; detail in Section 2).

02 Tile-sequencer implementation

Proposal - grounded in the current tree. Today's sequencer.sv is ~1522 LOC and carries a co-issue / RAW / SYNC interlock that the fully-exposed model removes.

Removed: the hardware interlock

The current sequencer serializes via a co-issue sync gate, a RAW hazard check, and SYNC_MXU. The fully-exposed model has none of these: the sequencer issues 1 instruction/cycle and never stalls except on flush.slot / halt. The compiler owns all hazards via the reservation table.

04 DMU: semantics and ports

Deferred · DMU internals

DMU internals (burst sizing, FIFO depth, dm_axi error/timeout, arbitration) are deferred and will be finalized during implementation. Only the semantics and the port interface are fixed here.

Semantics

  • Async DMA between devmem and SPAD; v1 2 slots (DMA_SLOT_COUNT=2).
  • dmu.load.spm = DRAM to SPAD; dmu.store.spm = SPAD to DRAM.
  • Descriptor = {SPAD row, devmem addr (literal-pool index; stride at pool[idx+1]), shape = (rows-1)<<8 | (cols-1)}.
  • On completion, sets a per-slot done bit that flush.slot waits on.
  • halt drains all outstanding transfers.
