    As-built, pre-merge

    This page tracks the real tree on origin/main plus the feature branches that implement the compute plane. Those branches are not yet merged; the integrated tile is their union, landed by milestone #16 (Section 6). Dispositions: keep, new, rewrite, cherry-pick.

    Version context: ISA 0.5.0 is the design target; the shipped SSOT (opcodes.py) is still 0.4.0 and the cutover (0.5.0 opcodes, PE_LATENCY 0 -> 2) is pending. See the ISA 0.5.0 status note.

    01 Overview and current tree¶

    The implementation maps the locked microarchitecture onto RTL modules. Module context (FIG. 1-2) is in the microarchitecture reference. The table below is the real npu/src/compute_tile/ tree with a disposition and source branch per file.

    Locked · sequencer decomposition

    The control front-end is pc.sv / pc_iram.sv / decoder.sv / dma_addr_gen.sv / perf_counters.sv (issues #204 / #180); the tile_sequencer.sv + loop_ctrl.sv / tile_agu_pkg.sv no-interlock FSM drives fetch / decode / issue on top of it (detail in Section 2).

    02 Tile-sequencer implementation¶

    The as-built tile_sequencer.sv + loop_ctrl.sv + tile_agu_pkg.sv (sequencer/feat-scaffold) implement the compute-tile control plane: fetch / decode / single-issue with no hardware interlock.

    Fully-exposed: no hardware interlock

    The sequencer issues one instruction per cycle and never stalls except on flush.slot / halt. There is no co-issue sync gate, no RAW hazard check, and no SYNC_MXU: the compiler owns all hazards via the reservation table (latency ledger in Section 6).

    modulefiledispositionrolekey interfaceFSM
    Tile sequencer tile_sequencer.sv new (sequencer/feat-scaffold) fetch -> decode -> issue 1/cy, never stalls except flush.slot / halt; per-unit issue bus (stubbed until wired) per-unit issue / decode buses IDLE -> FETCH (3-cyc pipe) -> ISSUE, no sync gate
    PC + decode + IRAM pc.sv + decoder.sv + pc_iram.sv keep (reuse, structural) PC, IRAM dual-port, field decode, literal-pool redirect mux IRAM 4096 x 64b dual-port; decoded fields out -
    Loop control loop_ctrl.sv new (sequencer/feat-scaffold) LIFO loop-stack (depth 4) + iv counters + bo[8] base-offset accumulators + stride apply loop.begin reads {hi, stride vector} from the pool; loop.end does iv+=step / bo+=stride / branch / pop push / iterate / pop
    Per-unit AGU tile_agu_pkg.sv new (sequencer/feat-scaffold) combinational agu_add: eff = operand_literal + bo[channel], no multiplier; channels BO_CH_X=0, W=1, B=2, YS=3 literal + bo in; effective address out -
    DMU dmu.sv deferred (last, #16) async DMA, 2 slots (DMA_SLOT_COUNT=2); tile_sequencer keeps DMU external via flush.slot / halt drain (Section 7) descriptor in; dm_axi master; SPAD ports; slot_done[] out per-slot
    • Fetch depth: the FETCH pipe (F1 / F2 / F3, ~3 cycles) is the physical realization of the single architectural F stage (a pipelined IRAM BRAM read), not an extra architectural stage: it still issues 1 instruction/cycle in steady state, matching the F → D → X → W issue model in the microarch reference.
    • Literal pool: reuse the existing IRAM-embedded pool machinery rather than a separate LPM RAM; the pool loads with the kernel image and is read in Decode via the redirect mux (see the microarch LPM card).
    • DMA outstanding: 2 slots (hwparams DMA_SLOT_COUNT=2).
    • Validated: test_tile_sequencer.py (straight-line one-issue no-stall, flush.slot spin, halt drain, illegal-op safe-stop) and test_loop_ctrl.py (4-deep LIFO, bo[8] stride accumulation vs the 32x32 GELU bo-oracle). Unit dispatch is stubbed at this stage.

    • Regenerate all: make -C npu generate-sv-pkgs (runs python3 scripts/constgen.py).
    • Selective: python3 npu/scripts/constgen.py --pkg config (or regmap / modes / isa / ip_params).
    • Staleness gate (CI): python3 npu/scripts/constgen.py --check (exit 1 if any output is stale).
    • #16 note: the PE_LATENCY=2 pipeline depth is a hwparams.py edit, then make -C npu generate-sv-pkgs. Do not hand-edit npu_config_pkg.sv; it is generated and the change would revert.

    06 Integration #16 and latency ledger¶

    Milestone #16 lands the union of the feature branches. It is not yet done. The integrated tile is rtl/feat-fp-pipeline (B, the pipelined fp core) with the row-pipelined vpu_ctrl, plus vreg_rf, lsu, tile_sequencer / loop_ctrl / tile_agu_pkg, and the X-from-VREG MXU datapath.

    Integration steps

    • MXU reconciliation: integrate B's timing (SKEW=3) with mxu-vreg-x's X-source. Hand-port the X-from-VREG datapath into B's mxu.sv; do not git-merge mxu-vreg-x's mxu.sv or sequencer.sv.
    • Compute-plane top: extend compute_tile.sv to wire tile_sequencer dispatch to the units + vreg_rf
    • spad + an exposed ACC read port; AGU resolve per operand via tile_agu_pkg::agu_add.
    • VPU on vreg_rf: per DECISION 1, the VPU uses the R_vpu read port + the shared single write port (time-shared with LSU). Required for the worked-example e2e (acc.store -> gelu -> vreg.store all on V16).
    • LSU-SPAD RL=2: align to real MEM_LATENCY=2, generalize rd_data_q to an RL-deep shift register (LSU latency 9 -> 10), re-run test_lsu.
    • DMU internals (dmu.sv) deferred to last; tile_sequencer keeps the DMU external (Section 7).

    Row-pipelining is PARTIAL

    Only 5 single-source ACC ops stream at 1 row/cycle within a tile: scale.bcast, scale.col.bcast, scale.imm, sub.bcast, add.col.bcast. gelu / exp2 / div / two-source / rowsum / rowmax stay latency-serial (deeper or multi-read, not sim==synth-safe to stream yet). The speedup is within-tile only (about 5.9x per tile), not cross-tile sustained throughput. The external start / done / bram_active contract is unchanged.

    07 DMU: semantics and ports¶

    Deferred to last (#16)

    DMU internals (burst sizing, FIFO depth, dm_axi error / timeout, arbitration) are deferred to the end of #16. Only the semantics and the port interface are fixed here; the tile_sequencer already keeps the DMU external via flush.slot / halt drain.

    Semantics¶

    • Async DMA between devmem and SPAD; v1 2 slots (DMA_SLOT_COUNT=2).
    • dmu.load.spm = DRAM to SPAD; dmu.store.spm = SPAD to DRAM.
    • Descriptor = {SPAD row, devmem addr (literal-pool index; stride at pool[idx+1]), shape = (rows-1)<<8 | (cols-1)}.
    • On completion, sets a per-slot done bit that flush.slot waits on.
    • halt drains all outstanding transfers.
