LSU 10-cycle pipeline

FIXED LATENCY = ROWS + SPAD_RD_LATENCY = 8 + 2 = 10 · one datapath, three ops · read row r on cycle r, write commits RL=2 later · owns the bf16 ↔ fp32 casts

addr-gen / issue read (SPAD RL=2 / async shim) per-element cast write

Reference (RTL-verified, lsu.sv)

  • FIXED LATENCY = ROWS + SPAD_RD_LATENCY = 8 + 2 = 10 cycles, uniform across all three ops. A read is issued for tile-row r on cycle r; the corresponding write commits RL = SPAD_RD_LATENCY = 2 cycles later.
  • One datapath. The SPAD read port has 2 cycles of registered read latency; the async ACC / VREG reads are pushed through an equal-depth (RL) shift register rd_data_sr, so all three ops share the same timing.
  • acc.store: ACC -> VREG, fp32, no cast. vreg.load: SPAD bf16 -> VREG fp32 widen. vreg.store: VREG fp32 -> SPAD bf16 narrow.
  • Whole-tile op: an [NxN] tile = N tile-rows, processed one 256-bit tile-row per cycle; fixed, data-independent latency (safe for the scoreboard-free sequencer). RL >= 1 generalizes; deployed RL = 2 matches spad.sv MEM_LATENCY = 2.

Depths are RTL-measured on rtl/full-npu-wire. Grounded in npu/src/compute_tile/lsu.sv and npu/src/npu/npu_config_pkg.sv (ROWS = N = 8, MEM_LATENCY = 2). Stage/latency view of the LSU sink.
